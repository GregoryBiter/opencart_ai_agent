# Инструкция по разработке и редактированию модулей (OpenCart 3)

Данный регламент предназначен для AI-агентов и разработчиков. Описывает процесс создания и модификации модулей с использованием локального окружения и утилиты `ocm`.

## 1. Подготовка окружения

- Репозиторий: используйте актуальный репозиторий целевого сайта для разработки и тестирования.
- Копирование инструкций: скопируйте содержимое этого репозитория (все файлы, кроме `.git`) в корень вашего сайта.
- Основная рабочая область: `dev-modules/` — в ней создаются папки отдельных модулей.

## 2. Структура модуля в dev-modules

Для каждого модуля соблюдайте структуру:

```
dev-modules/
└── [название_модуля]/
  ├── upload/        # Файлы модуля (структура OpenCart)
  ├── install.xml    # OCMOD (если требуется)
  └── README.md, sql и т.д.
```

Файлы внутри `upload/` должны соответствовать структуре OpenCart (например, `upload/catalog/controller/extension/module/...`).

## 3. Процесс разработки

- Новые файлы: добавляйте контроллеры, модели и шаблоны только в `dev-modules/[имя]/upload/`.
- Правки ядра: все изменения ядра описывайте в `install.xml` как OCMOD — не редактируйте файлы ядра напрямую.
- Редактирование: правки вносятся исключительно в папке модуля; синхронизация выполняется через `ocm`.

## 4. Интеграция и тестирование (CLI)

Используйте утилиту `ocm` для синхронизации модуля с рабочей копией сайта:

- Первичная установка:

```bash
ocm install [название_модуля]
```

Команда копирует или создаёт симлинки из `dev-modules` в рабочие папки сайта.

- Режим разработки (watch):

```bash
ocm dev [название_модуля]
```

Этот режим автоматически обновляет файлы на сайте при изменениях в папке модуля.

## 5. Золотые правила (Best Practices)

- Изоляция: не работайте в `system/storage/modification/` напрямую.
- Логи OCMOD: после установки/обновления модификатора проверяйте логи в админ-панели на ошибки (NOT FOUND и пр.).
- OCMOD: используйте уникальные фрагменты в `<search>` и атрибут `trim="true"`; предпочтительны `position="before"` или `position="after"` вместо `replace`.
- Безопасность: после разработки проверьте соответствие новых путей и прав доступа требованиям безопасности OpenCart.
